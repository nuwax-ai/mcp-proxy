## 简介

本指南旨在帮助新用户快速搭建并运行 `mcp-proxy` 项目。项目包含三个核心组件：`mcp-proxy`（MCP代理服务）、`document-parser`（文档解析服务）和 `voice-cli`（语音转文字服务）。通过本指南，您将学会如何克隆仓库、配置环境、启动服务，并完成从添加MCP插件到接收SSE消息的完整流程。

## 环境准备

在开始之前，请确保您的系统满足以下基本要求：

- **操作系统**：Linux、macOS 或 Windows
- **磁盘空间**：至少 1GB 可用空间
- **网络连接**：能够访问 GitHub、crates.io 和 PyPI（或配置国内镜像源）
- **Git**：已安装并可执行

### 推荐开发环境
- **编辑器**：VSCode（推荐安装 rust-analyzer、crates、YAML 等插件）
- **终端**：支持 UTF-8 编码的现代终端（如 iTerm2、Windows Terminal）

## Rust工具链与依赖安装

### 1. 安装 Rust 工具链

```bash
# 下载并安装 rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 激活环境（或重启终端）
source $HOME/.cargo/env

# 验证安装
rustc --version
cargo --version
```

### 2. 安装构建工具

```bash
# 安装 cargo-generate（用于项目模板）
cargo install cargo-generate

# 安装 git-cliff（用于生成 changelog）
cargo install git-cliff

# 安装 typos（拼写检查）
cargo install typos-cli
```

### 3. 安装 Python 依赖（document-parser）

进入 `document-parser` 目录并初始化 Python 虚拟环境：

```bash
cd document-parser
document-parser uv-init
```

该命令会自动创建 `./venv` 虚拟环境并安装 `mineru` 和 `markitdown` 等依赖。

### 4. 安装 Docker（可选，用于构建）

```bash
# 确保 Docker 已安装并运行
docker --version
docker-compose --version
```

## 服务启动方式

### 1. 使用 Makefile 启动（推荐）

`Makefile` 提供了便捷的构建和运行命令：

```bash
# 查看所有可用命令
make help

# 构建 document-parser（x86_64 Linux）
make build-document-parser-x86_64

# 构建 voice-cli（x86_64 Linux）
make build-voice-cli-x86_64

# 构建所有组件
make build-all-x86_64

# 构建并运行 Docker 镜像
make build-image
make run
```

### 2. 使用 Cargo 直接启动

#### 启动 MCP Proxy 服务

```bash
# 在项目根目录
cargo run -p mcp-proxy
```

默认监听端口：`8086`

#### 启动 Document Parser 服务

```bash
# 在 document-parser 目录
cd document-parser
document-parser server
```

默认监听端口：`8087`

#### 启动 Voice CLI 服务

```bash
# 在 voice-cli 目录
cd voice-cli
cargo run --bin voice-cli -- server run
```

默认监听端口：`8080`

## 配置文件说明

### 1. Document Parser 配置 (`document-parser/config.yml`)

```yaml
server:
  port: 8087
  host: "0.0.0.0"

mineru:
  backend: "pipeline"  # 可选: vlm-sglang-engine (GPU加速)
  max_concurrent: 1

markitdown:
  enable_plugins: false

storage:
  oss:
    access_key_id: "${OSS_ACCESS_KEY_ID}"
    access_key_secret: "${OSS_ACCESS_KEY_SECRET}"
```

### 2. MCP Proxy 配置 (`mcp-proxy/config.yml`)

```yaml
server:
  port: 8086
log:
  level: debug
  path: logs
```

### 3. Voice CLI 配置 (`voice-cli/config.yml`)

```yaml
server:
  port: 8080
  max_file_size: 209715200  # 200MB

whisper:
  default_model: "base"
  models_dir: "./models"
  auto_download: true
```

### 4. 环境变量配置

创建 `.env` 文件以避免硬编码敏感信息：

```bash
# .env
OSS_ACCESS_KEY_ID=your_access_key
OSS_ACCESS_KEY_SECRET=your_secret_key
```

## 端到端使用流程

### 1. 添加 MCP 插件

1. 在 MCP 社区查找所需插件（如 MySQL 插件）
2. 复制其 JSON 配置
3. 使用 `/mcp/sse/check_status` 接口注册插件：

```bash
curl -X POST http://localhost:8086/mcp/sse/check_status \
  -H "Content-Type: application/json" \
  -d '{
    "mcpId": "mysql-test",
    "mcpJsonConfig": "{\"mcpServers\": {\"mysql\": {\"command\": \"go-mcp-mysql\", \"args\": [\"--host\", \"127.0.0.1\", \"--user\", \"root\", \"--pass\", \"password\", \"--port\", \"3306\", \"--db\", \"test\"]}}}",
    "mcpType": "Persistent"
  }'
```

### 2. 建立 SSE 连接

服务启动后，可通过 SSE 协议接收实时数据：

```bash
curl "http://localhost:8086/mcp/sse/proxy/mysql-test/sse" \
  -H "x-mcp-json: <your_mcp_json_config>" \
  -H "x-mcp-type: Persistent"
```

### 3. 发送消息

向 MCP 服务发送查询请求：

```bash
curl -X POST http://localhost:8086/mcp/sse/proxy/mysql-test/message \
  -H "Content-Type: application/json" \
  -H "x-mcp-json: <your_mcp_json_config>" \
  -H "x-mcp-type: Persistent" \
  -d '{
    "id": "query_1",
    "method": "completions.create",
    "params": {
      "messages": [
        {
          "role": "user",
          "content": "查询 users 表中的所有记录"
        }
      ]
    }
  }'
```

## 示例代码与API调用

### 1. 使用 examples 目录

项目包含多个示例代码，位于各组件的 `examples/` 目录：

- `document-parser/examples/markdown_image_processing.rs`：演示 Markdown 图像处理

### 2. 核心 API 调用方式

#### Document Parser API

```bash
# 上传并解析 PDF 文件
curl -X POST http://localhost:8087/api/v1/documents/parse \
  -F "file=@sample.pdf" \
  -F "format=pdf"
```

#### Voice CLI API

```bash
# 语音转文字
curl -X POST http://localhost:8080/transcribe \
  -F "audio=@speech.mp3" \
  -F "model=base"
```

## 故障排除

### 常见问题

#### 1. 虚拟环境未激活

```bash
# Linux/macOS
source ./venv/bin/activate

# Windows
.\venv\Scripts\activate
```

#### 2. 依赖安装失败

```bash
# 使用国内镜像源
uv pip install -i https://pypi.tuna.tsinghua.edu.cn/simple/ mineru[core]
```

#### 3. 端口被占用

```bash
# 查找占用进程
lsof -i :8086

# 终止进程
kill -9 <PID>
```

#### 4. GPU加速不生效

- 确保安装了 `sglang[all]`
- 配置 `mineru.backend: "vlm-sglang-engine"`
- 参考 [CUDA_SETUP_GUIDE.md](file://document-parser/CUDA_SETUP_GUIDE.md)

### 获取帮助

```bash
# 查看详细帮助
document-parser troubleshoot
voice-cli --help

# 查看日志
tail -f document-parser/logs/log.$(date +%Y-%m-%d)
```
