## 引言
本文档全面介绍`CacheManager`的设计与实现，重点描述其基于文档指纹的缓存键生成机制与多级缓存架构。详细说明缓存条目生命周期管理、LRU或TTL过期策略的实现方式，以及内存压力下的自动清理机制。分析其在重复解析相同文档或相似内容时的性能增益，并阐述缓存一致性保障措施，防止脏数据问题。提供缓存命中率监控指标及容量规划建议。

## 缓存键生成机制

缓存管理器采用基于文档指纹的缓存键生成机制，通过哈希算法为每个文档生成唯一标识。该机制确保相同内容的文档始终映射到相同的缓存键，实现高效的缓存命中。

```mermaid
flowchart TD
Start([开始]) --> HashInput["输入文档标识符"]
HashInput --> CreateHasher["创建DefaultHasher实例"]
CreateHasher --> HashData["对标识符进行哈希计算"]
HashData --> FormatKey["格式化为十六进制字符串"]
FormatKey --> AddPrefix["添加'doc:'前缀"]
AddPrefix --> ReturnKey["返回最终缓存键"]
ReturnKey --> End([结束])
style Start fill:#f9f,stroke:#333
style End fill:#f9f,stroke:#333
```

## 缓存生命周期管理

缓存条目通过TTL（Time To Live）机制管理生命周期，确保数据的时效性和内存的有效利用。

```mermaid
stateDiagram-v2
[*] --> Created
Created --> Active : "插入缓存"
Active --> Expired : "created_at.elapsed() > ttl"
Active --> Removed : "显式删除"
Expired --> Removed : "自动清理"
Removed --> [*]
note right of Active
每次访问时更新last_accessed
增加access_count计数
end note
note right of Expired
is_expired()方法检测
下次访问时触发清理
end note
```

## 驱逐策略实现

系统支持多种缓存驱逐策略，可根据配置灵活选择最适合业务场景的算法。

```mermaid
graph TD
Start([开始]) --> CheckPolicy["检查驱逐策略"]
CheckPolicy --> LRU{策略=LRU?}
LRU --> |是| EvictLRU["移除访问顺序中最旧的条目"]
CheckPolicy --> LFU{策略=LFU?}
LFU --> |是| EvictLFU["移除访问频率最低的条目"]
CheckPolicy --> FIFO{策略=FIFO?}
FIFO --> |是| EvictFIFO["移除最早插入的条目"]
CheckPolicy --> Random{策略=Random?}
Random --> |是| EvictRandom["随机移除一个条目"]
EvictLRU --> UpdateStats["更新驱逐统计"]
EvictLFU --> UpdateStats
EvictFIFO --> UpdateStats
EvictRandom --> UpdateStats
UpdateStats --> End([结束])
```

## 自动清理机制

系统通过后台任务定期清理过期缓存，确保内存使用效率和数据新鲜度。

```mermaid
sequenceDiagram
participant CleanupTask as "清理任务"
participant DC as "DocumentCache"
participant RC as "ResultCache"
participant MC as "MetadataCache"
participant Stats as "CacheStats"
loop 每5分钟
CleanupTask->>DC : remove_expired()
DC-->>CleanupTask : 返回过期数量
CleanupTask->>RC : remove_expired()
RC-->>CleanupTask : 返回过期数量
CleanupTask->>MC : remove_expired()
MC-->>CleanupTask : 返回过期数量
alt 有过期条目
CleanupTask->>Stats : record_expired_cleanup(count)
end
end
Note over CleanupTask : 清理间隔由配置决定<br/>默认为5分钟
```
